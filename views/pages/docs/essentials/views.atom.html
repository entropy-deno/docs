Views

Entropy comes with handy built-in view template engine. View templates in your app are placed in /views directory and inside modules. View files have the *.atom.html extension.

Template engine

Entropy template engine allows you to create loops, conditionals and variable interpolation.

An example view template may look like this:

@raw 
        

          

          

          


          
views/home.html

        
<main>
  @for (post in posts)
    <article class="post">{{ post.content }}</article>
  @empty
    <p>There are no posts yet.</p>
  @/for

  @if (!user)
    <h2>You have to log in.</h2>
  @/if
</main>
@/raw

Conditional blocks

You can specify a conditional block using special @raw@if@/raw directive

@raw 
        

          

          

          


          
views/home.html

        
<nav>
  @if (user.logged)
    <div>{{ user.name }}</div>
  @else
    <p>You are not logged in.</p>
  @/if
</nav>
@/raw

Form request methods

Entropy lets you to use all HTTP methods in forms thanks to @raw @method @/raw directive. Just pass a method name and you'll be able to use PUT, PATCH, DELETE and other methods in HTML forms.

@raw 
        

          

          

          


          
views/home.html

        
<form action="/login" method="post">
  @method('PATCH')
</form>
@/raw

JSON serialization

Sometimes you need to pass some data from backend to frontend using HTML <script /> tag. You can accomplish that by converting data to JSON with @raw @json @/raw directive:

@raw 
        

          

          

          


          
views/home.html

        
<script>
  window.userData = @json(userData);
</script>
@/raw

Partials

Entropy provides support for partials. You can split your view into smaller pieces using @raw @include /@raw directive:

@raw 
        

          

          

          


          
views/home.html

        
<main>
  @include('partials/content');
</main>
@/raw

CSRF token

For every user session Entropy generates a unique token to protect your application from Cross-Site Request Forgery attacks.

Anytime you define HTML forms with method other than GET and HEAD, you need to add a hidden field containing generated token. Otherwise, you won't be able to pass the form and you'll get the 419 Invalid Token error.

To add the token field just use the @raw @csrf @/raw directive:

@raw 
        

          

          

          


          
views/home.html

        
<form action="/login" method="post">
  @csrf
</form>
@/raw

Raw template

Sometimes you may want to left some parts of code uncompiled. For example, when you are using some frontend framework, you may want not to parse the code with template engine. That's why Entropy comes with a @raw @raw @/raw directive:

@raw 
        

          

          

          


          
views/home.html

        
@​raw
  {{ content }}
@​/raw
@/raw

Switch

Just like switch statements in JavaScript and TypeScript, you can use them in your views:

@raw 
        

          

          

          


          
views/home.html

        
<main>
  @switch (role)
    @case ('user')
      <p>You are a user</p>
    @/case

    @case ('admin')
      <p>You are an administrator</p>
    @/case

    @case ('moderator')
      <p>You are a moderator</p>
    @/case

    @default
      <p>Invalid role</p>
    @/default
  @/switch
</main>
@/raw

@raw @dev and @prod @/raw

Entropy comes with two directives which allow you to render content only in development or production environment:

@raw 
        

          

          

          


          
views/home.html

        
@dev
  <script src="/build.development.js"></script>
@/dev

@prod
  <script src="/build.production.js"></script>
@/prod
@/raw

@raw @stack and @push @/raw

You can push view blocks to named stacks which will be rendered anywhere in your template using @raw @stack and @push @/raw directives:

@raw 
        

          

          

          


          
views/home.html

        
<head>
  ...

  @stack('scripts')
</head>
        

          

          

          


          
views/home.html

        
@push('scripts')
  <script src="/lib.js"></script>
@/push

<!-- Somewhere else (child view, etc.): -->

@push('scripts')
  <script src="/app.js"></script>
@/push
@/raw

@raw @hotReload @/raw

You are able to use Hot Reload (auto refresh) in your view files. To enable it, you need to add @raw @hotReload @/raw directive to your <head> tag:

@raw 
        

          

          

          


          
views/home.html

        
<head>
  ...

  @hotReload
</head>
@/raw
